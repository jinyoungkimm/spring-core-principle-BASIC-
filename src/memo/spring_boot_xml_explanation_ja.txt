【質問】
スプリングブートを使っていると「XMLベースのBean設定をあまり使わない」と言われますが、
それはどういう意味ですか？

【回答】 これは「XMLが使えない」という意味ではありません。
正確には次のような意味です。

Spring Bootでは、従来XMLで行っていた多くの設定を、
Javaコード・アノテーション・自動設定（Auto Configuration）で
ほぼ置き換えることができるようになりました。

【Spring Bootが登場する前のSpring】
・Bean定義や依存関係、DataSourceなどを大量のXMLで設定によりXMLファイルが肥大化し、保守や理解が難しい
・プログラミング言語でないXMLはコンパイラー過程がないため、誤字などによる設定エラーを未然に防止できない
・設定とアプリケーションコードが完全分離されてるため、コードの全体像が把握しづらい
→Spring Bootの登場により上記の問題は払拭される

Spring Bootでは以下の仕組みにより、XML設定を書く必要がほとんどなくなりました。
・アノテーションベース設定（@Component, @Service, @Repository など）
・Java Config（@Configuration + @Bean）
・application.yml / application.properties による外部設定
・Auto Configuration による自動Bean登録

そのため「XML設定を使わない」という表現は、
「XMLが禁止されている」のではなく、
「使う理由がほとんどなくなった」という意味になります。

現在でもレガシーシステムや一部ライブラリでは
XML設定が使われることはありますが、 Spring Bootで新規開発を行う場合は、
XMLを使わないのが事実上の標準です。

【それでもXMLを使う時】
・大規模なレガシープロジェクトのBean設定情報の一度に変更が困難な時
・外部ライブラリがXML設定を要する時
