必ずコンストラクタ注入を使用しよう

・実務では、アプリケーションの起動から終了まで依存関係が変更されることはほとんどない
（むしろ、変更されるべきではない）。

このような理由から、
コンストラクタはオブジェクト生成時に一度だけ呼び出されるという特性を利用し、
依存関係を「不変（immutable）」にしよう。

・テストコード作成時に、開発者のミスによる依存関係注入の漏れを防ぐことができる。
・Springフレームワークに依存せず、純粋な Java コードでテストコードを容易に作成できる。
→ 実務では、純粋な Java でテストコードを作成するケースが非常に多い。
